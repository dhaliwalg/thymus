Thymus

An immune system for your codebase

A Claude Code plugin that learns your project's architectural rules and enforces them on every edit.

View on GitHub

The Problem

Claude Code writes great code. But it doesn't remember your architecture between sessions. Over days, patterns drift — boundaries get crossed, conventions break, dependencies leak where they shouldn't. By the time you notice, the debt has compounded.

How It Works

  1. 1

    /thymus:baseline

    Scans your project, detects modules, boundaries, and patterns. Proposes invariants. You review and save.

  2. 2

    Every edit, automatically

    A PostToolUse hook checks each file change against your rules in under two seconds. Warns Claude immediately if something drifts.

  3. 3

    /thymus:scan

    Full-project scan on demand. Catches violations the hooks might miss.

  4. 4

    /thymus:health

    Generates an HTML health report with score, trend sparklines, and debt projections.

What It Enforces

Boundaries — Database access only through the repository layer

db.query() found outside src/repos/ — violation

Patterns — No raw SQL outside the db module

SELECT * found in src/handlers/user.py — violation

Conventions — Tests colocated with source files

test_auth.py expected next to auth.py — missing

Dependencies — Axios only used in the API client

import axios found in src/components/ — violation

Zero dependencies. Bash + jq. Installs in seconds.
